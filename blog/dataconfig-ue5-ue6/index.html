DataConfig UE5 and Beyond

2026-06-19

Epic Games pushed UE 5.8.0 yesterday along with the UE6 announcement. We pushed the latest 1.7.6 DataConfig and updated DataConfig JSON Asset with UE 5.8 support. So what's next for DataConfig? Here's the rough plan:

  • DataConfig is considered done in development. That is, we consider all features we ever wanted in this project to have been implemented. If you find this concept novel, you'd be amazed that there's a versioning scheme called Kelvin Versioning that counts releases down to 0, after which there are no more releases.
  • UE 5.8 is likely the last major UE release. If that's the case DataConfig 1.7.6 would be the final codebase that works [from UE 4.25 to UE 5.8].
  • When UE6 is out, we're likely to freshen up the codebase to only work in UE6. This is because we already have tons of #ifdefs to support multiple UE versions, and it'll be unmanageable with UE6 added. That said, there are likely no new major features to be added, so newer releases will have feature parity with 1.7.6. One of the reasons we spend the effort to support all these engine versions is that we want a clean and reliable way to get your UE data in and out of the engine.

And that's it. See you in the future! (or maybe just when 5.9 hits).
